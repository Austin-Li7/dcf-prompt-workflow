# Review Rubric

## Primary Standard

Practical usefulness over formal benchmark purity.

## Scoring Dimensions

1. Catch power
   Can the validation prompt detect the kinds of errors it is supposed to catch?

2. False-positive burden
   Does the prompt over-flag issues that are likely acceptable or easily explained?

3. Reviewer usability
   Is the output concise and actionable enough for a human approver?

4. Rollout readiness
   Is the prompt stable enough to reuse across additional companies without manual prompt surgery?

## Recommended Labels

- `works well`
- `usable with fixes`
- `high false-positive risk`
- `under-checks important issues`
- `not enough evidence yet`
